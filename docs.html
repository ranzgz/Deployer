POST /api/deploy

This endpoint allows you to deploy a static website by uploading files and providing configuration as multipart/form-data.

Parameters (multipart/form-data)

Field Name Type Description
subdomain String The desired project name/subdomain. Required
domainId Number The ID of the domain to use, fetched from /api/domains. Required
domainName String The full name of the domain selected (e.g., "rafxz.xyz"). Required
files File / [File] One or more files for the website. The entry point must be index.html. You can also upload a single .zip file which will be automatically extracted by the client. Required

Example Usage (cURL)

Here's how you can call the API using cURL from your terminal:

curl --location --request POST 'https://[your-deployer-url].vercel.app/api/deploy' \ --header 'Content-Type: multipart/form-data' \ --form 'subdomain="my-new-site"' \ --form 'domainId="1"' \ --form 'domainName="rafxz.xyz"' \ --form 'files=@"/path/to/your/project.zip"'

Responses

On Success (200 OK)

{ "message": "Deployment successful! Domain is now verifying.", "finalUrl": "my-new-site.rafxz.xyz" }

On Error (4xx/5xx)

{ "message": "Error message describing what went wrong." }
