POST /api/deploy

This endpoint allows you to deploy a static website by uploading files and providing configuration as multipart/form-data.

Parameters (multipart/form-data)

Field Name Type Description
subdomain String The desired project name/subdomain. Required
domainId Number The ID of the domain to use, fetched from the API. Required
domainName String The full name of the domain selected (e.g., "rafxz.xyz"). Required
files File / [File] One or more files for the website. index.html is required. You can also upload a single .zip file. Required

Example Usage

curl -X POST \
  https://deployer.fishnemo.xyz/api/deploy \
  -H 'Content-Type: multipart/form-data' \
  -F 'subdomain=my-new-site' \
  -F 'domainId=1' \
  -F 'domainName=rafxz.xyz' \
  -F 'files=@"/path/to/your/project.zip"'
import requests

url = "https://deployer.fishnemo.xyz/api/deploy"
file_path = '/path/to/your/project.zip'

payload = {
    'subdomain': (None, 'my-new-site'),
    'domainId': (None, '1'),
    'domainName': (None, 'rafxz.xyz')
}
files = {
    'files': ('project.zip', open(file_path, 'rb'), 'application/zip')
}

response = requests.post(url, data=payload, files=files)

print(response.status_code)
print(response.json())
const fs = require('fs');
const FormData = require('form-data');
const axios = require('axios');

const deploy = async () => {
  const url = 'https://deployer.fishnemo.xyz/api/deploy';
  const filePath = '/path/to/your/project.zip';

  const form = new FormData();
  form.append('subdomain', 'my-new-site');
  form.append('domainId', '1');
  form.append('domainName', 'rafxz.xyz');
  form.append('files', fs.createReadStream(filePath));

  try {
    const response = await axios.post(url, form, {
      headers: {
        ...form.getHeaders()
      }
    });
    console.log('Status:', response.status);
    console.log('Data:', response.data);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
};

deploy();

Responses

✅ On Success (200 OK)

{
  "message": "Deployment successful! Domain is now verifying.",
  "finalUrl": "my-new-site.rafxz.xyz"
}

❌ On Error (e.g., 400 Bad Request)

Returned if required fields are missing.

{
  "message": "Missing required fields or files."
}

❌ On Error (e.g., 500 Internal Server Error)

Returned for server-side issues, like an invalid API key.

{
  "message": "Vercel Deploy Error: The provided authorization token is invalid."
}
